# Frontend Quickstart (Beacon)

This workspace uses a PNPM monorepo. The frontend app is `@workspace/beacon` in `artifacts/beacon`.

## Prerequisites

- Node.js 20+ (24 works as well)
- PNPM (via Corepack recommended)

```bash
corepack enable
corepack prepare pnpm@latest --activate
```

## Install Dependencies

Always install from the workspace root (`Asset-Manager`), not from subfolders.

```bash
cd /Users/natemacbook/Desktop/intex_final/Asset-Manager
pnpm install
```

## Run Frontend Locally

From `Asset-Manager`:

```bash
pnpm dev
```

This runs:

```bash
pnpm --filter @workspace/beacon dev
```

Vite will print a local URL (usually `http://127.0.0.1:5173` or next available port).

## Build Frontend

```bash
pnpm --filter @workspace/beacon build
```

Build output:

`artifacts/beacon/dist/public`

## Environment Variables

Frontend API base URL:

- `VITE_API_BASE_URL` (example: `https://api.yourdomain.com`)

If not set, frontend calls relative paths like `/api/...`.

## Vercel Deployment Notes

- Vercel root directory: `Asset-Manager`
- `vercel.json` is already configured with:
  - install command
  - frontend build command
  - output directory
  - SPA route rewrite to `index.html`

Set this env var in Vercel:

- `VITE_API_BASE_URL`

## Common Issues

- **Installed from wrong directory:** Run `pnpm install` from `Asset-Manager` root.
- **Rollup darwin module errors:** Reinstall from root (`pnpm install --no-frozen-lockfile`) after config changes.
- **Seeing `SIGINT` on shutdown:** Normal when stopping dev server with `Ctrl+C`.
- **Port in use:** Vite auto-selects another port.
